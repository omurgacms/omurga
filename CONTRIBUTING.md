# Contributing to Omurga CMS

Thank you for considering a contribution to Omurga CMS.

## How to contribute

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Test the installation and admin panel.
5. Open a pull request with a clear description.

## Contribution areas

- Core bug fixes
- OMG theme examples
- PHP theme examples
- Block examples
- Package examples
- Documentation
- UI improvements
- Translation improvements

## Basic rules

- Do not include `config.php` in commits.
- Do not include `storage/installed.lock` in commits.
- Do not include generated cache or log files.
- Do not include old release zip files inside `packages/`.
- Do not break existing content, themes or builder data.
- Keep changes simple and easy to review.

## Pull request checklist

- [ ] PHP syntax checked
- [ ] Installation still works
- [ ] Admin panel opens without white screen
- [ ] Default themes still work
- [ ] README or docs updated if needed

# Omurga CMS'ye Katkı Sağlama

Omurga CMS'ye katkı sağlamak istediğiniz için teşekkürler.

## Nasıl katkı verilir?

1. Repoyu fork edin.
2. Yeni bir geliştirme dalı oluşturun.
3. Değişikliklerinizi yapın.
4. Kurulum ve admin panelini test edin.
5. Açıklayıcı bir pull request gönderin.

## Katkı alanları

- Çekirdek hata düzeltmeleri
- OMG tema örnekleri
- PHP tema örnekleri
- Blok örnekleri
- Paket örnekleri
- Dokümantasyon
- Arayüz iyileştirmeleri
- Dil iyileştirmeleri

## Temel kurallar

- `config.php` commit etmeyin.
- `storage/installed.lock` commit etmeyin.
- Cache ve log dosyalarını commit etmeyin.
- Eski sürüm ziplerini `packages/` içine koymayın.
- Mevcut içerikleri, temaları veya builder kayıtlarını bozmayın.
- Değişiklikleri sade ve incelenebilir tutun.
